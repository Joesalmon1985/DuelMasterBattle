# Duel Master Battle — Project Plan

## Goal
Build a playable **Human vs Bot** wizard duel in Godot 4.4.x: encounter-driven rules (1–4 points, filtered magic pools, variable attack limits), alternating turns, backed by tested pure logic in Python and GDScript with shared golden fixtures.

## Primary success condition
Launch Godot → select **encounter** → **Start duel** → set pattern via click-slot picker (pool-filtered) → **Cast pattern** → alternate attacks (human first) with visible dual feedback histories → result → **New duel** or **Back to menu**.

**Archmage Duel** must match classic Mastermind: 4 points, 10 magic types, 12 attacks, Expert solver.

## Tech stack
| Layer | Choice |
|-------|--------|
| Rules (Python) | `python_prototype/duel_mastermind/` + pytest |
| Rules (Godot) | `godot_project/sim/` + headless `run_tests.gd` |
| Encounter config | `DuelRuleset` / `DmbDuelRuleset` + `encounters.py` / `encounters.gd` |
| Python UI | tkinter (minimal proof; may lag Godot feature set) |
| Godot UI | Godot 4.4.x Control scenes, `EncounterSession` autoload |
| Parity | `shared_fixtures/` JSON golden files |
| Bot | Ruleset-driven RandomBot + SolverBot (difficulty per encounter) |

## Architecture
```
shared_fixtures/     ← generated by Python, consumed by both
python_prototype/    ← pure logic + minimal tkinter
godot_project/
  sim/               ← authoritative rules, encounters, bots (headless)
  client/            ← UI scenes (non-authoritative)
tools/               ← test runners, cleanup scripts
docs/                ← rules, encounters, playtest, milestones
```
