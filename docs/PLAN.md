# Duel Master Battle — Project Plan

## Goal

Build a **mobile-first real-time wizard ward duel** in Godot 4.4.x: Human vs rival wizard with independent cast windows, encounter-driven rules (1–8 loci), global difficulty selection, and tested pure logic in Python and GDScript.

**Source of truth:** [PRD.md](PRD.md) — supersedes the earlier alternating-turn prototype model.

## Primary success condition

Launch Godot (portrait mobile) → select **difficulty** → select **encounter** → set ward pattern → real-time duel with cast windows, aggregate Fracture/Echo/Fade feedback, barrier animations → result (Victory / Defeat / Clash / Stalemate) → retry or menu.

**Archmage Duel** remains the balance anchor: 4 loci, 10 essences, 12 attacks.

## Tech stack

| Layer | Choice |
|-------|--------|
| Rules (Python) | `python_prototype/duel_mastermind/` + pytest |
| Rules (Godot) | `godot_project/sim/` + headless `run_tests.gd` |
| Real-time sim | `DmbRealtimeDuelSim` (authoritative) |
| Encounter config | `DmbDuelRuleset` + `encounters.gd` |
| Difficulty | `DmbDifficultyProfile` (separate from encounter) |
| Godot UI | Portrait Control scenes, `EncounterSession` autoload |
| Parity | `shared_fixtures/` JSON golden files |
| Bot | Ruleset + difficulty-driven RandomBot / SolverBot |

## Architecture

```
shared_fixtures/     ← generated by Python, consumed by both
python_prototype/    ← pure logic + realtime_duel.py
godot_project/
  sim/               ← authoritative rules, realtime sim, bots (headless)
  client/            ← UI scenes + animation layer (non-authoritative)
tools/               ← test runners, export scripts
docs/                ← PRD, rules, encounters, milestones
```
